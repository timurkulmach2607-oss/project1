# Changelog - Deep Audit Fixes (30.04.2026)

## [11.45.20] - 30.04.2026

### Додано
- **М'яке видалення користувачів:** Реалізовано механізм `is_active` у таблиці `users`. Користувачі більше не видаляються фізично, що дозволяє зберігати історію логів.
- **Whitelist для редагування:** Додано захист від несанкціонованої зміни критичних колонок у Дашборді. Дозволено редагувати лише `status`, `assigned_to` та `description`.

### Виправлено
- **Масова реєстрація:** Виправлено логіку `quantity`. Тепер при додаванні декількох одиниць створюються окремі записи в БД з унікальними серійними номерами та штрихкодами.
- **Генерація серійних номерів:** Реалізовано автоматичне додавання суфіксів (`-1`, `-2`) для серійників при масовому додаванні обладнання та унікальні UUID для витратних матеріалів.
- **NaN значення при імпорті:** Додано очищення CSV через `.fillna('')`, що усуває помилку відображення тексту "nan" замість порожніх полів.
- **Помилки фільтрації Pandas:** Додано примусове приведення типів `.astype(str)` у пошуку по Дашборду для запобігання `TypeError`.
- **Стабільність фото:** Реалізовано використання абсолютних шляхів (`os.path.abspath`) для відображення фото активів.

### Оптимізація
- **Керування кешем:** Впроваджено автоматичне очищення `docx_cache` при зміні будь-яких даних на Дашборді, що гарантує актуальність документів для завантаження.
- **UI фрагменти:** Посилено стабільність вкладки сканування через використання фрагментів Streamlit для запобігання помилок `removeChild`.
